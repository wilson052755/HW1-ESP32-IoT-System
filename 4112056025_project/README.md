# AIoT DHT11 即時溫濕度監測系統

## 專案大綱
本專案為一個完整的物聯網 (IoT) 數據收集與視覺化系統。透過前後端分離設計，硬體設備可將感測到的溫濕度透過 WiFi 上傳至電腦伺服器，並以動態、具備互動性的網頁儀表板即時呈現。

主要模組包含：
1. **硬體感測端 (`DHT11_WiFi.ino`)**：負責讀取真實 DHT11 溫濕度，並透過 WiFi 模組傳送 HTTP GET 請求。
2. **模擬器腳本 (`simulate_dht11.py`)**：在無硬體時提供虛擬亂數數據供系統測試。
3. **Flask 後端 API (`flask_api.py`)**：輕量級的 HTTP 伺服器，負責接收感測器傳來的數據並安全寫入資料庫。
4. **SQLite 自動化資料庫 (`aiotdb.db`)**：自動建立與儲存包含時間戳記的溫濕度數據表 (`sensors`)。
5. **Streamlit 動態儀表板 (`app.py`)**：提供客製化深色系主題，並搭配精細的互動式 Altair 折線圖（支援滑鼠游標追蹤與提示框）來做數據走勢的即時繪製。

## 操作方法與說明

### 系統環境要求
第一次執行前，請確保已安裝以下 Python 必備套件：
```bash
pip install streamlit pandas altair flask
```

### 啟動前置準備
不論以下哪種方式，請先開啟命令提示字元 (cmd) 並切換至專案根目錄：
```bash
cd c:\Users\user\Desktop\DIC4
```

### 方式一：真實硬體運作 (建議)
1. **啟動後端 API 接收伺服器**：
   執行以下指令，讓電腦在 `5000` port 等待接收硬體資料：
   ```bash
   python flask_api.py
   ```
2. **啟動即時監控儀表板**：
   請開啟**第二個 cmd 視窗**執行以下指令，系統會自動在瀏覽器彈出監視網頁：
   ```bash
   python -m streamlit run app.py
   ```
3. **配置 Arduino 程式並燒錄硬體**：
   - 開啟 `DHT11_WiFi.ino`，填寫您的 `ssid` (WiFi 名稱) 與 `password` (WiFi 密碼)。
   - 將 `serverName` 內的 `192.168.X.X` 修改為您目前電腦在區網內的 IP 位址。
   - 將程式燒錄至您的 ESP8266 / ESP32 等具有 WiFi 的開發板，並插電運行即可！

### 方式二：無硬體模擬運作 (純軟體測試體驗)
若手邊暫時沒有真實硬體設備，可使用軟體腳本模擬硬體拋送行為：
1. **啟動感測模擬腳本**：
   該腳本會每 2 秒向 SQLite 資料庫產生一筆虛擬資料。
   ```bash
   python simulate_dht11.py
   ```
2. **啟動即時監控儀表板**：
   請開啟**第二個 cmd 視窗**執行：
   ```bash
   python -m streamlit run app.py
   ```

## 介面亮點與特色
- **客製化莫蘭迪灰紫主題**：告別刺眼白光，帶來兼具沉穩、高品質閱讀體驗的深色視覺環境。
- **高互動的游標追蹤系統**：內建滑鼠游標追蹤鉛直線，當移至圖表上時自動高亮該時間點，並浮現精確時間與溫濕度數值的互動提示樞紐。
- **儀表板控制區**：右側選單內建資料重撈頻率的調節滑桿，讓您針對監測需求靈活動態調整。
